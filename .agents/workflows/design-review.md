# Design Review Workflow

Use this workflow when the task is UI/UX critique, visual polish, copy refinement, or demo readiness.

## Inputs

- Current screen or component.
- User goal.
- Demo audience.
- Any visual reference.

## Steps

1. Read `.agents/ui-ux-designer.md`.
2. Identify the screen's main job.
3. Audit clarity, layout, alignment, density, copy, and visual style.
4. Check whether the UI reinforces Program Pulse as a visibility layer.
5. Identify what should be removed, renamed, or simplified.
6. Hand implementation notes to the Frontend Developer if code changes are needed.

## Output Format

```text
Design intent

What works

What is confusing

Recommended changes

Copy/label changes

Implementation notes

Demo impact
```

## Checks

- Are repeated cards aligned?
- Is the typography hierarchy clear?
- Are labels concise?
- Are buttons compact?
- Are long strings handled?
- Does evidence remain central?
- Is human review obvious?
- Does the UI avoid inflated dashboard noise?
